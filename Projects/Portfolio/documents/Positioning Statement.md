# About. 포지셔닝 선언문

### 마케팅 개념의 포지셔닝 선언문이란

포지셔닝 선언문은 '우리의 제품/서비스는 특정 목표 고객에게 경쟁사 대비 어떤 독특하고 의미 있는 가치를 제공하는가'를 명확하게 요약한 내부 전략 문서로, 브랜드의 정체성과 모든 마케팅 활동의 핵심 방향을 제시하는 '브랜드의 헌법'과 같다.

이는 고객의 마음속에 특정 이미지를 각인 시켜 경쟁 제품과 차별화하고, 고객이 우리 브랜드를 선호하는 이유를 명확히 설명하는 역할을 한다.

#### 주요 구성 요소 및 특징

- 목표 고객 : 누굴 위하 제품인가? (예: 바쁜 워킹맘, 기술에 민감한 얼리어답터)
- 제품/서비스 카테고리: 무엇을 제공하는가? (예: 프리미엄 유기농 이유식, AI 기반 일정 관리 앱)
- 핵심 가치/차별점: 경쟁사보다 무엇이 더 나은가? (예: 시간 절약, 더 높은 안정성, 독보적인 기술)
- 내부 지침: 광고나 메시지 개발의 기준이 됨 (단순한 슬로건이 아님)

# 개발자 포트폴리오를 위한 포지셔닝 선언문

### 1. 포지셔닝 선언문을 개발자 관점으로 바꾸면?

앞에서 봤듯, 원래(브랜드/제품)는 "우리 제품은 누구에게, 어떤 상황에서 경쟁사 대비 어떤 독특한 가치를 제공하는가?" 가 포지셔닝의 관점이다.

하지만 개발자의 관점, 즉 포트폴리오에 적용해보면..

- 나는 어떤 팀/회사에게,
- 어떤 문제 상황에서,
- 다른 개발자와 비교했을 때 어떤 차별적 가치를 제공하는가?

즉, **자기 소개 != 포지셔닝 선언문** ,
포지셔닝은 **설명이 아니라 판단을 유도하는 선언**이다.

## 개발자 포지셔닝 선언문의 핵심 목적

- 이 사람은 어떤 개발자인가 (X)
- 이 사람은 우리 팀에 필요할까? (O)

즉, **면접관이 읽고 생각ㅎ지 않아도 자동으로 판단하게 만드는 문장**

## 개발자용 포지셔닝 선언문의 4가지 구성 요소

### 1. 목표 '고객' → 나를 필요로 하는 팀

> **목표 고객 :**
> 누구를 위한 제품인가? (예: 바쁜 워킹맘, 기술에 민감한 얼리어답터)

브랜드의 목표 고객 -> 개발자에겐 이것

- 어떤 팀?
  - 스타트업, 프로덕트 팀, UX 중심 팀
- 어떤 상황?
  - 기능이 많아서 UI가 복잡해진 팀
  - 사용자가 이탈 원인을 못 찾는 팀
  - 흐름 정리가 필요한 서비스

예를 들자면, **"나는 어떤 문제를 가진 팀에서 가장 강점이 살아나는가?"**

### 2. 제품/서비스 → 내 역할과 포지션

> **제품/서비스 카테고리 :**
> 무엇을 제공하는가? (예: 프리미엄 유기농 이유식, AI 기반 일정 관리 앱)

브랜드의 제품 -> 이게 개발자의 역할이다.

- 프론트엔드 개발자
- UI 구현자 (X)
- 문제 해결 방식이 있는 FE 개발자 (O)

즉, **"무슨 기술을 쓰는지"보다, "무슨 판단을 하는 개발자인지"**

### 3. 핵심 가치/ 차별점 → 다른 지원자와의 차이

> **핵심 가치/차별점 :**
> 경쟁사보다 무엇이 더 나은가? (예: 시간 절약, 더 높은 안정성, 독보적인 기술)
> 이게 제일 중요함.

- 흔한 차별화
  - 사용자의 경험을 중요하게 생각합니다.
  - 구조적으로 분석합니다.
  - 기록을 꾸준히 합니다.
- 좋은 차별화
  - 문제를 화면 단위가 아니라 흐름 단위로 본다.
  - UI를 만들기 전에 사용자의 이전 선택을 먼저 추적한다.
  - 버그보다 사용자 망설임 지점을 먼저 의심한다.

즉, **"같은 문제를 받았을 때 다른 FE랑 내가 다르게 보는 지점은 뭘까?"** 를 생각해보자.

### 4. 내부 지침 → 포폴 전체를 관통하는 기준

> **내부 지침 :**
> 광고나 메시지 개발의 기준이 됨 (단순한 슬로건이 아님)

브랜드 헌법 -> 개발자 포폴의 기준이 된다.

- About
- 프로젝트 설명
- 기술 선택 이유
- 회고 글
  전부를 관통하는 판단 기준이 되어야한다.

예를 들면, "나는 기능을 추가할 때맏 UI를 늘리기보다 사용자가 어디서 멈출지를 먼저 본다."

이 문장이 있으면 프로젝트 설명도 달라지고, UI 설명도 달라지고, 회고도 달라진다.

### 개발자 포지셔닝 선언문의 성격 정리

| 항목 | 포지셔닝 선언문            |
| ---- | -------------------------- |
| 목적 | 채용 판단을 빠르게 유도    |
| 톤   | 설명 x / 선언 o            |
| 대상 | 나 자신 x / 면접관 o       |
| 위치 | 포폴 첫 화면, About 최상단 |
| 역할 | “이 사람은 이런 애다” 각인 |

  - 결과: 불필요한 UI를 줄이고 망설임을 줄이는 경험
