# 목표 중심(Goal-oriented) 과 사고 중심(Thinking-oriented)란

목표 중심 사고와 사고 중심 사고는 일의 접근 방식과 초점에 있어서 명확한 차이가 있다. 목표 중심 사고는 최종 결과 달성에 중점을 두는 반면, 사고 중심 사고는 과정 자체의 분석과 생각에 더 무게를 둔다. 더 자세히 알아보도록 하자

> **🤔 먼저 한 줄 정의**
>
> 목표 중심: "무엇을 만들고 싶은가"에서 출발하는 사고
>
> 사고 중심: "왜 이런 문제가 생겼는가"에서 출발하는 사고
>
> 즉 차이점은 출발점이다.

## 목표 중심 사고 (Goal-Oriented Thinking)

- **초점:** 달성하고자 하는 **최종 결과물** 이나 성과.

### 특징

- 원하는 결과를 구체적이고 객관적인 목표로 설정한다.
- 목표 달성을 위한 효율적인 절차와 방법을 수단으로 활용한다.
- 결과를 통해 성과를 평가하며, 성공과 실패가 명확하게 구분된다.
- 조직 관리나 새로운 프로젝트 생성에 적합할 수 있으며, 높은 집중력과 추진력을 발휘하게 한다.
- 단점으로는 목표 달성에 실패했을 때 좌절감을 느끼거나, 과정의 즐거움을 놓칠 수 있다는 점이 있다.

즉.. 목표 중심의 FE는 보통 이렇게 말한다.

- "디테일한 UI를 만들고 싶다."
- "사용자 경험을 개선하고 싶다."
- "더 직관적인 화면을 만들고 싶다."

이 말의 공통점은 **결과물이 먼저 있음**

### 구조로 보면..

```scss
원하는 결과(목표)
- 그에 맞는 방법을 찾음
- 구현
```

즉, 예를 들면 "사용자 경험이 좋아야 하니까 애니메이션을 넣자", "디테일이 중요하니까 여백을 더 조정하자." 이런게 나쁘다는게 아닌, 대부분의 FE가 이 방식을 사용한다.

-> 즉, 포폴에 적기엔 너무나도 평범하게 들려서 매리트가 없다.

## 사고 중심 사고 (Thought-Oriented Thinking)

- **초점 :** 목표 달성 과정에서 발생하는 **생각, 분석, 의미 등** 내면의 정신 활동 자체

### 특징

- 어떤 행동을 하기 전에 상황을 분석하고, 심사숙고하는 경향이 있다.
- 현실이나 타인으로부터 한 걸음 물러나 객관적으로 상황을 파악하려고 한다.
- 자신의 위치를 정한 후에 행동을 촉발하기 때문에 신중한 의사결정이 가능하다.
- 결과보다는 일의 의미와 과정을 중시ㅣ하며, 이는 개인의 성장이나 내적 ㅁ나족감에 기여할 수 있다.
- 애니어그램과 같은 성격 유형 이론에서 "머리(사고)중심" 유형의 사람들이 이러한 특성을 보인다.

즉, 사고 중심 FE는 출발이 아예 다르다고 생각하면 된다.

- "왜 여기서 사용자가 멈췄지?"
- "왜 이 버튼에서 이탈이 생기지?"
- "왜 이 화면에서 다시 돌아가지?"

**문제의 원인** 부터 본다고 생각하면 된다.

### 구조로 보면

```scss
문제 상황 관찰
- 원인 가설
- 흐름 분석
- 해결
- 결과로 UI 디테일이 생김
```

즉, 예를 들면 "여기서 사용자가 망설이는 이유는 선택자가 동시에 너무 많이 보이기 때문이네"

-> 그래서 애니메이션을 줄이고, 레이아웃을 단순화 하고, CTA위치를 바꾼다.

- 여기서 중요한 포인트는 "디테일을 낳자"가 목표가 아니라는 것이다. 디테일은 문제 해결의 결과일뿐..

## 예를 들어보자

> 문제: **회원가입 도중 이탈률이 높다.**

여기서 **목표 중심 FE는..**

"회원가입 UX를 개선해야겠다."
"디테일을 살려야겠다."

- 입력창 애니메이션 추가
- 버튼 hover 효과 추가
- 컬러 강조

결과: **예쁘긴 한데.. 이탈 원인이 해결됐는지는 불명확**

그리고 **사고 중심의 FE는..**

"왜 여기서 나갈까?"

- 필드 수가 한 화면에 너무 많음
- 다음 단계가 불 명확함
- 에러 메시지가 늦게 나옴

**해결:**

- 단꼐를 나눔
- 다음 행동을 명확히 함
- 에러를 즉시 보여줌

**결과:**

- UI는 더 단순해짐
- 디테일은 줄었는데 UX는 좋아짐

## 마치며

| 구분      | 목표 중심      | 사고 중심        |
| --------- | -------------- | ---------------- |
| 출발점    | 만들고 싶은 것 | 생긴 문제        |
| 디테일    | 목적           | 결과             |
| UI        | 감각/취향      | 판단/논리        |
| 설명 방식 | 가치 설명      | 문제 해결 과정   |
| 포폴 인상 | 무난           | “아 얘는 다르네” |

즉, 정리하자면 목표 중심의 FE는 "어떤 UI를 만들고 싶은지"를 말하고, 사고 중심의 FE는 "왜 그런 UI가 나올 수밖에 없었는지"를 말한다.
