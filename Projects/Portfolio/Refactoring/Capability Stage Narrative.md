# Capability Stage Narrative

# Overview

Capability Stage는 단순한 기술 소개 섹션이 아니다.

이 Stage의 목적은:

- 어떤 사고방식으로 경험을 설계하는지
- 왜 이런 인터랙션 구조를 선택하는지
- 사용자의 흐름을 어떻게 바라보는지

를 하나의 scroll narrative 안에서 경험시키는 것이다.

핵심 방향성은 Apple 스타일의 pinned storytelling 구조를 기반으로 한다.

# IntroScene

## 목적

“나는 어떤 철학으로 경험을 설계하는 사람인가”를 선언하는 구간.

단순 자기소개가 아니라:

- 경험 중심 사고
- 흐름 중심 설계
- 구조와 인터랙션에 대한 관점

을 강하게 인지시키는 역할.

---

## Core Narrative

### 고정되는 것

```
FLOW.
STRUCTURE.
MOTION.
```

```
경험을 설계하는 세 가지 방식.
```

### 스크롤에 따라 교체되는 것

- 설명
- 철학
- 시각적 사례

즉:

```
상단 선언 = 고정
하단 정보 = 점진적 reveal
```

구조.

---

# PHASE 01 — 선언

## 역할

Capability Stage 전체의 철학적 방향 제시.

## 핵심 메시지

```
저는 단순히 화면을 구현하는 것보다,
사용자가 어떤 흐름으로 이해하고 행동하는지를 더 중요하게 바라봅니다.

그래서 인터페이스,
인터랙션,
상태 흐름,
그리고 구조까지도
하나의 연결된 경험처럼 설계하려고 합니다.
```

---

## 전달해야 하는 인상

- 단순 UI 제작자가 아님
- 경험 흐름을 설계하는 사람
- 구조와 인터랙션을 연결해서 사고함

---

# PHASE 02 — 철학

## 역할

“좋은 경험이란 무엇인가”에 대한 관점 설명.

---

## 핵심 메시지

```
좋은 경험은
흐름을 의식하지 않게 만듭니다.

사용자는 기능을 하나씩 읽기보다,
흐름 안에서 자연스럽게 이해하고 행동한다고 생각합니다.

그래서 저는
설명을 늘리는 것보다,
망설임 없이 이어질 수 있는 구조를 더 중요하게 바라봅니다.
```

---

## 핵심 관점

사용자는:

- 기능을 읽지 않는다
- 흐름 안에서 행동한다
- 자연스럽게 이해한다

따라서 중요한 것은:

```
망설임 없이 이어지는 구조
```

를 만드는 것.

---

# PHASE 03 — 철학의 시각적 증명

## 역할

앞에서 설명한 철학을 실제 감각으로 보여주는 파트.

즉:

```
그래서 어떤 경험을 만들었는데?
```

에 대한 시각적 답.

---

## 핵심 개념

캐릭터를 단순 decoration이 아니라:

```
사용자의 시선과 감정 흐름을 조율하는 장치
```

로 사용한다.

---

## 이 파트에서 증명하는 것

- 시선 유도
- 심리적 부담 완화
- 자연스러운 행동 흐름
- 감정적 연결

즉:

```
왜 이 경험이 이런 구조여야 하는가
```

를 설명하는 장면.

---

# ExperienceCapabilityScene

## 목적

“그래서 나는 어떤 경험을 가능하게 하는가”를 설명하는 구간.

중요한 점은:

```
기술이 아니라 경험 가치 중심으로 설명해야 한다.
```

---

# 01. Structure, behind simplicity.

## 핵심 메시지

```
복잡함은 줄이고,
흐름은 더 자연스럽게.
```

---

## 전달하려는 인상

- 읽기 쉬운 구조
- 유지보수 가능한 흐름
- 협업 가능한 설계
- 책임이 명확한 구조

---

## 포함 요소

- monorepo architecture
- reusable components
- DTO ↔ UI separation
- trigger ownership
- naming consistency
- role-based flow

---

## 핵심 관점

```
사용자 경험은
화면만으로 만들어지는 것이 아니라,
내부 구조의 안정성에서도 나온다.
```

---

# 02. AI, as a creative partner.

## 핵심 메시지

```
AI를 단순 생성 도구가 아니라,
사고를 확장하는 파트너로 사용한다.
```

---

## 01. Comparative Reasoning

### 질문하고 비교하는 사고 방식

```
답을 받아들이기보다,
더 나은 방향을 계속 비교합니다.
```

---

## 02. Expanded Exploration

### 탐색 범위를 넓히는 방식

```
하나의 도구보다,
더 넓은 탐색 과정을 중요하게 생각합니다.
```

---

## 03. Understanding Over Answers

### 결과보다 이해를 우선하는 태도

```
빠르게 만드는 것보다,
깊게 이해하는 과정을 중요하게 생각합니다.
```

---

## 이 섹션의 핵심

단순히:

```
AI를 잘 사용한다
```

가 아니라:

```
AI를 활용해 사고 범위를 확장한다
```

를 보여주는 것.

---

# 03. Visual sense, beyond interfaces.

## 목적

기능을 넘어서:

- 분위기
- 리듬
- 감각
- 인상

까지 설계하려는 태도를 보여주는 섹션.

---

## 핵심 메시지

```
저는 단순히 기능을 배치하기보다,
사용자가 화면을 어떻게 느끼게 될지까지
함께 고민하려고 합니다.
```

---

## 보여줄 요소

- 직접 그린 캐릭터
- visual exploration
- typography rhythm
- motion balance
- composition
- visual storytelling

---

## 중요한 이유

여기서 처음으로:

```
이 사람은 감각까지 설계하려고 하는구나
```

라는 인상을 강하게 남기게 된다.
