# 배열의 프로토타입과 배열 조작 메서드

> **배열이란?**
>
> 배열은 객체이지만, 정수 타입인 인덱스를 프로퍼티로 갖는 특별한 데이터이다.

## 1. 배열의 프로토타입 체인

```js
const arr = [];
```

배열은 객체 리터럴로 생성한 객체와 달리 이들의 객체는 독특하게 각자 자신의 프로토타입을 따로 정의하고 있다.

즉, `Array.prototype`을 직접 프로토타입으로 가진다.

```js
Object.getPrototypeOf(arr) === Array.prototype;

// true
```

배열의 전체 프로토타입 체인은 다음과 같다.

```
arr
 ↓
Array.prototype
 ↓
Object.prototype
 ↓
null
```

`Array.prototype`에는 배열의 전용 메서드가 정의되어 있다.

```js
Array.prototype.push;
Array.prototype.pop;
Array.prototype.map;
Array.prototype.filter;
```

배열이 `push()`, `pop()`, `map()` 등을 사용할 수 있는 이유는 배열 객체가 직접 이 메서드를 가진 것이 아니라, 프로토 타입 체인을 따라 `Array.prototype` 에서 찾아 사용하기 때문이다.

```js
const arr = [];

arr.hasOwnProperty("push");
// false

Array.prototype.hasOwnProperty("push");
// true
```

## 2. 배열의 `length` 프로퍼티

`length`는 배열의 길이를 나타내는 프로퍼티이다.

```js
const arr = [1, 2, 3];

console.log(arr.length);
// 3
```

### 가장 큰 인덳를 기준으로 계산된다.

배열이 길이는 단순히 실제 원소 개수만 세는 것이 아니라 다음 기준으로 계산된다.

```
가장 큰 인덱스 + 1
```

```js
const arr = [];

arr[0] = 1;
arr[2] = 3;

console.log(arr);
// [1, empty, 2]

console.log(arr.length);
// 3
```

`arr[1]`에는 값이 없지만 가장 큰 인덱스가 2이므로 length는 3이다.

```
최대 인덱스 2 + 1 = length 3
```

값이 비어 있는 부분을 **빈 슬롯** 이라고 한다.

### 배열을 수정하면 `length`도 자동으로 변한다.

```js
const arr = [];

console.log(arr.length);
// 0

arr.push(1);

console.log(arr.length);
// 1
```

배열에 원소를 추가하거나 삭제하면 length도 자동으로 갱신된다.

### length의 길이

1. **`length`를 줄이면 뒤쪽 원소가 삭제된다.**

```js
const arr = [1, 2, 3, 4];

arr.length = 2;

console.log(arr);
// [1,2]
```

2. **반대로 `length`를 늘리면 빈 슬롯이 생긴다.**

```js
const arr = [1, 2];

arr.length = 4;

console.log(arr);
// [1, 2, empty x 2]
```

### 빈 슬롯과 희소 배열

```js
const arr = [1, , 3];
```

- **인덱스 1 :** 빈 슬롯
- **arr :** 희소 배열

> 빈 슬롯은 특정 인덱스가 존재하지 않는 상태이고, 희소 배열은 빈 슬롯을 하나 이상 가진 배열이다.

빈 슬롯과 `undefined`는 다르다.

```js
const sparse = [1, , 3];
const normal = [1, undefined, 3];

1 in sparse; // false
1 in normal; // true
```

- 빈 슬롯: 인덱스 프로퍼티 자체가 없음
- undefined: 인덱스는 있고 값이 undefined
  희소 배열의 빈 슬롯은 메서드마다 다르게 처리된다.

- forEach(), map(), filter() : 이들은 빈 슬롯을 건너뛴다.
- find(), findIndex() : 이들은 빈 슬롯도 undefined로 보고 탐색한다.

이처럼 동작이 일관적이지 않으므로 특별한 경우가 아니라면 희소 배열 사용을 피하는 것이 좋다.

## 3. 원본 배열을 변경하는 메서드

| 메서드      | 동작                | 반환값           |
| ----------- | ------------------- | ---------------- |
| `push()`    | 뒤에 원소 추가      | 새로운 길이      |
| `pop()`     | 마지막 원소 삭제    | 삭제한 원소      |
| `unshift()` | 앞에 원소 추가      | 새로운 길이      |
| `shift()`   | 첫 원소 삭제        | 삭제한 원소      |
| `splice()`  | 원소 추가·삭제·교체 | 삭제한 원소 배열 |
| `sort()`    | 원소 정렬           | 정렬된 원본 배열 |

```js
const arr = [1];

arr.push(2, 3);

console.log(arr);
// [1,2,3]
```

`sort()` 는 기본적으로 문자열 기준으로 정렬한다.

```js
const arr = [3, 2, 21];

arr.sort();

console.log(arr);
// [2, 21, 3]
```

숫자를 정렬하려면 비교 함수를 전달한다.

```js
arr.sort((a, b) => a - b);
```

## 4. 새로운 배열을 반환하는 메서드

| 메서드     | 동작                                     |
| ---------- | ---------------------------------------- |
| `concat()` | 배열이나 값을 합친 새 배열 반환          |
| `slice()`  | 특정 범위를 복사한 새 배열 반환          |
| `map()`    | 각 원소를 변환한 새 배열 반환            |
| `filter()` | 조건을 만족하는 원소만 담은 새 배열 반환 |

```js
const arr = [1, 2, 3];

const newArr = arr.slice(1, 3);

console.log(arr);
// [1, 2, 3]

console.log(newArr);
// [2, 3]
```

## `slice()`와 `splice()`

| 구분         | `slice()`      | `splice()`     |
| ------------ | -------------- | -------------- |
| 역할         | 배열 일부 복사 | 추가·삭제·교체 |
| 원본 배열    | 변경하지 않음  | 변경함         |
| 두 번째 인수 | 종료 인덱스    | 삭제 개수      |

> `slice()`는 복사하고, `splice()`는 원본 배열을 수정한다.

## 얕은 복사

`slice()`와 스프레드 연산자는 새로운 배열을 만들지만, 내부 객체의 참조는 그대로 공유한다.

```js
const user = {
  name: "김봄",
};
const arr = [user];
const copied = [...arr];

console.log(arr === copied);
// false

console.log(arr[0] === copied[0]);
// true
```

배열 자체는 다르지만 내부 객체는 같은 객체를 가르킨다.

### 핵심 정리

- 배열은 `Array.prototype`을 프로토타입으로 가지며, 배열 메서드는 프로토타입 체인을 통해 찾아 사용한다.
- `length` 는 가장 큰 인덱스에 1을 더한 값이다.
- 빈 슬롯은 인덱스 프로퍼티 자체가 없는 상태이고, 이를 가진 배열을 희소 배열이라고 한다,
- 배열 메서드는 원본 배열을 변경하는지, 새로운 배열을 반환하는지 구분해서 사용해야한다.
