# [OOP]디자인 패턴(Design Pattern)이란? - 장점 및 종류

## 한줄 정의

디자인 패턴은 개발하면서 발생하는 **반복적인 문제** 들을 어떻게 해결할 것인지에 대한 **해결 방안** 으로 실제 현업에서 비즈니스 요구 사항을 프로그래밍으로 처리하면서 만들어진 다양한 해결책 중에서 많은 사람들이 인정한 **모범 사례(Best Practice)** 이다.

대표적인 객체 지향 디자인 패턴은 객체 지향의 주요 특성인 **캡슐화, 상속, 추상화, 다형성** 과 **SOLID 설계 원칙** 을 활용해 구현된다.

단, 디자인 패턴은 완성된 코드를 그대로 복사하는 것이 아니라, 특정 설계 문제를 해결하기 위해 객체의 역할과 관계를 어떻게 구성할지 제시하는 설계 방법이다.

## 왜 사용하는가?

프로그래밍을 하다 보면 다음과 같은 비슷한 설계 문제가 반복해서 발생한다.

- 객체를 어떤 방식으로 생성할 것인가?
- 객체의 상태 변화를 다른 객체에 어떻게 전달할 것인가?
- 기능이나 알고리즘을 쉽게 교체하려면 어떻게 설계해야 하는가?
- 기존 코드들을 크게 수정하지 않고 새로운 기능을 추가하려면 어떻게 해야 하는가?
- 서로 다른 인터페이스를 가진 객체를 어떻게 연결할 것인가?

이러한 문제를 개발자가 매번 처음부터 해결하면 비슷한 시행착오가 반복된다.
디자인 패턴은 여러 개발자가 실제 개발 과정에서 고민하고 검증한 해결 방법을 제공하여, 설계 과정의 시행착오를 줄이는 데 도움을 준다.

단 디자인 패턴은 완성된 코드를 그대로 복사하는 것이 아니다. 특정 상황에서 객체와 책임을 어떻게 구성할지 알려주는 **설계 아이디어** 다.

## OOP란?

OOP는 **`Object-Oriented Programming`** 의 약자로, 한국어로는 **객체 지향 프로그래밍** 이라고 한다.

객체 지향 프로그래밍은 프로그램을 데이터와 기능을 가진 객체들의 집합으로 보고, 객체들이 서로 협력하도록 구성하는 프로그램이 방식이다.

객체는 일반적으로 다음 두 요소를 가진다.

- **상태 :** 객체가 가지고 있는 데이터
- **행동 :** 객체가 수행할 수 있는 기능

```ts
class User {
    constructor(private name: string) {}

    introduce() {
        return `안녕하세요. 저는 ${this.name}입니다.`;
    }

}
```

위 코드에서 `name` 은 **객체의 상태**이고,
`introduce()`는 **객체의 행동**이다.
