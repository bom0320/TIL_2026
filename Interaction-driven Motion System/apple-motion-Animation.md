# 애플 웹사이트 애니메이션

애플 웹사이트의 부드러운 스크롤 애니메이션을 구현하는 핵심 코딩 기술은 크게 세가지로 요약된다.

## 1. 캔버스 이미지 스퀸스 (Canvas Image Sequence)

비디오 파일을 그대로 올리면 사용자의 스크롤 속도에 맞춰 정밀하게 제어하기 어렵고, 로딩이 무겁다.
대신 애플은 수백 장의 **고화질 정지 이미지(PNG/WebP)**를 준비한다.

### 작동 방식 :

3D 툴로 렌더링한 수백 개의 프레임(주로 PNG 또는 WebP 형식)을 서버에서 미리 불러온다. 사용자가 스크롤하면 JS가 현재 스크롤 위치를 계산해 순서의 이미지를 `<canvas>` 앨리먼트에 즉시 그려준다.

### 고급진 포인트 :

일반 비디오는 스크롤에 맞춰 뒤로 감거나 특정 시점으로 이동할 때 끊김(Jank)이 발생할 수 있지만, 캔버스 방식은 프레임 단위로 완벽하게 제어되므로 **스크롤과 1:1로 동기화된** 극강의 부드러움을 보여준다.

## 2. 스크롤 트리거와 라이브러리 (GSAP & ScrollTrigger)

복잡한 타임라인(예: 스크롤에 따라 텍스트를 사라지고 제품은 커지는 효과)를 조율하기 위해 전문 라이브러리를 주로 사용한다.

### GSAP (GreenSock Animation Platform):

웹 애니메이션의 표준과 같은 라이브러리로, 매우 정밀한 성능을 제공한다.

### ScrollTrigger:

특정 요소가 화면에 들어올 때 애니메이션을 시작하거나, 스크롤 막대의 움직임을 애니메이션의 진행률과 1:1로 연결(scrub)하는 기능을 수행한다.

## 3. CSS Sticky & 계산 기술

화면의 레이아웃을 잡는 데는 다음과 같은 속성들이 필수적이다.

### `position: sticky` :

- 특정 섹션에 도달했을 때 제품이나 배경이 화면에 고정되어 머무르게 한다.
- 사용자가 스크롤을 계속해도 요소는 제자리에 있고 내부 애니메이션만 진해오디는 구조를 만든다.

### 가속도 및 보간법 (Interpolation) :

- 사용자의 스크롤은 투박할 수 있지만, 코드로 이전 **값과 현재 값 사이를 부드럽게 채워주는 계산을 추가**하여 '미끄러지는 듯한' 특유의 조작감을 만든다.

## 4. 인터랙티브 레이어링 (Interactive Layering)

제품 이미지 위에 텍스트나 다른 요소들이 겹쳐지며 나타나는 효과이다.

### Sticky & Fixed:

- 특정 섹션에 도달하면 제품 이미지를 화면 중앙에 고정(Sticky) 시키고, 그 위로 텍스트 레이어만 위아래로 지나가게 하여 깊이감(Depth)를 만든다.

### 마스킹 (Masking):

- 글자 뒤로 제품이 비치거나, 스크롤에 따라 원형 마스크가 커지면서 새로운 화면에 나타나는 효과(Clip-path)를 적극적으로 사용한다.

## 5. 3D 웹 렌더링 (선택 사항)

최근에는 이미지 시퀸스를 넘어 웹 브라우저에서 직접 3D 모델을 돌리는 방식을 쓰기도 한다.

- **`Three.js`** : 웹에서 3D 객체를 구현하는 대표적인 라이브러리로, 아이폰이나 맥북의 3D 모델을 직접 브라우저에 띄우고 스크롤에 다라 각도를 조절할 수 있게 해준다.

# 고급진 정체성을 만드는 '한 끗'

기술보다 중요한 것은 '의도된 움직임'이다.

- **의미 있는 모션 :** 모든 애니메이션은 제품의 특장점을 설명하기 위해 존재한다. 칩셋 성능을 설명할 때 칩 내부로 줌인되는 식이다.
- **가속도의 자연스러움 :** 물리 법칙을 무시한 급격한 멈춤이 없다. 관성이 느껴지도록 설계된 **이징(Easing) 값을 통해 사용자가 '디지털 기기를 조작하고 있다는'** 느낌을 준다.

즉, 고급스러운 웹사이트는 단순히 '화려한 기술'만으로 완성되지 않는다. 애플처럼 **"정체성(Identity)"** 이 느껴지는 웹사이트를 만들고 싶다면, 기술적인 구현 이전에 다음 3단계 전략을 고민해보도록 하자.

## 1. 브랜드의 '정체성' 시각화하기 (기획 단계)

애플이 애니메이션을 쓰는 이유는 "우리는 정밀하다"는 메시지를 전달하기 위해서이다. 본인이 만들고자 하는 사이트의 핵심 메시지는 무엇인가?

### 핵심 가치 정의:

- 신뢰감(금융), 혁신(태크), 부드러움(뷰티/패션) 등 한 단어를 정하자

### 시그니처 움직임:

- 신뢰감이라면 딱딱 끊어지는 정확한 움직임을, 부드러움이라면 서서히 나타나는(Fade-in) 효과를 메인으로 잡자

### 여백의 미(White Space):

- 고급스러움의 80%는 여백에서 나온다. 콘텐츠를 꽉 채우지 말고, 사용자가 애니메이션에 집중할 수 있는 공간을 확보하자

## 2. '디테일'의 한 끗 차이 (디자인 단계)

애플 스타일의 핵심은 **'물리 법칙'** 을 따르는 자연스러움이다.

### 이징(Easing) 함수:

- 애니메이션이 시작하고 끝날 때 속도가 일정하면 촌스럽다.
- `power4.out` 같은 가속도 곡선을 사용하여, 처음엔 빠르고 끝은 아주 부드럽게 멈추도록 설정하자.

### 고화질 소스:

- 아무리 코딩을 잘해도 이미지가 깨지면 저렴해보인다.
- 시퀸스 이미지나 배경은 반드시 고해상도(Retina 대응)로 준비해야 한다.

### 마이크로 인터렉션:

- 버튼에 마우스를 올렸을 때 살짝 커지거나, 배경색이 미세하게 변하는 등의 작은 효과들이 모여 '고급진' 느낌을 만든다.

## 3. 기술적 로드맵 (구현 단계)

### 1) 레이아웃:

- `position: sticky` 를 활용해 화면을 고정하고 스크롤해도 배경이 유지되는 구조를 먼저 만든다.

### 2) 라이브러리 도입:

- GSAP(GreenSock)과 ScrollTrigger를 넣는다. 애플의 스타일을 가장 쉽고 강력하게 구현 간으하다.

### 3) 성능 최적화:

- 애니메이션이 끊기면(Lag) 고급스러움이 단번에 깨진다. `requestAnimationFrame` 같은 브라우저 최적화 기술을 이해하거나, 이미지 용량을 최소화하는 작업이 필수이다.
