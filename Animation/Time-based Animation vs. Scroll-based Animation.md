# Time-based Animation vs. Scroll-based Animation(Scrub)

프론트에드에서 애니메이션을 만들다 보면 이런 고민을 하게 된다.

> "페이지 들어오자마자 자동으로 재생되는 애니메이션이랑 스크롤에 따라 움직이는 애니메이션.."
> 결국 눈에 보이는 건 비슷한 거 같은데, 굳이 나눌 필요가 있을까?

결론부터 말하면 **겉보기엔 비슷해보여도, UX의 설계 개념은 완전히 다르다.**

이 글은 그 차이를 정리하느 기록이다.

## 1. 애니메이션은 '무엇을 기준으로 움직이느냐'로 나뉜다.

프론트엔드 애니메이션은 크게 두 가지로 나눌 수 있다.

### 1) Time-based Animation (시간 기반 애니메이션)

> **정해진 시간(duration)에 따라 재생되는 애니메이션**

- 시작하면 끝날 때까지 시간 기준으로 진행
- 사용자의 행동과 무관
- 영상처럼 "보여주는 연출"

**예시**

- Hero 인트로 애니메이션
- 페이지 진입 시 타이틀 등장
- 무한 반복 애니메이션(wave, marquee)

```ts
gsap.from(".title", {
  y: 40,
  opacity: 0,
  duration: 1.2,
});
```

### 2) Scroll-based Animation (스크롤 기반 애니메이션, Scrub)

> 스크롤 진행도에 따라 애니메이션이 변하는 방식

- 애니메이션 진행률 = 스크롤 위치
- 사용자가 멈추면 애니메이션도 멈춤
- 위로 스크롤하면 되감기 됨
- 사용자가 "연출을 완성"

**예시**

- 스크롤할수록 채워지는 텍스트
- 섹션에 진입하면서 서서히 드러나는 타이틀
- 요소드이 모여드는 인터렉션

```ts
gsap.to(".fill", {
  opacity: 1,
  scrollTrigger: {
    trigger: ".section",
    scrub: true,
  },
});
```

## 2. "결국 똑같아 보이는데?"라는 착각이 생기는 이유

특히 이런 구조에서 헷갈리기 쉽다.

- 한 페이지에 여러 섹션이 있고
- 페이지에 진입하자마자 About 섹션이 보이고
- 자동 재생 애니메이션과 ScrollTrigger가 **같은 시점에서 시작될 때**

이 경우 **'시작 순간만 보면'** 두 방식이 같아보인다.

하지만 이는 **출발선만 같은 것이지, 진행 방식은 전혀 다르다.**

## 3. 차이가 드러나는 실제 사용자 행동

### 사용자가 스크롤을 멈췄을 때

- Time-based:
  - 애니메이션은 계속 진행됨
- Scroll-based:
  - 애니메이션도 멈춤

### 사용자가 빠르게 스크롤해서 스쳐 지나갈 때

- Time-based:
  - 연출을 놓칠 수 있음
- Scroll-based:
  - 스쳐간 만큼만 연출됨 -> 자연스러움

### 사용자가 위아래로 스크롤을 반복할 때

- Time-based:
  - 이미 끝난 애니메이션
- Scroll-based:
  - 다시 채워지고 다시 비워짐

**사용자 개입 가능 여부가 가장 큰 차이점이다.**

## 4. UX 관점에서의 차이

| 기준        | Time-based | Scroll-based |
| ----------- | ---------- | ------------ |
| 기준        | 시간       | 스크롤       |
| 사용자 제어 | 없음       | 있음         |
| 멈춤/되감기 | 불가       | 가능         |
| 체감        | 영상       | 인터랙션     |
| 역할        | 연출       | 반응         |

## 5. 언제 어떤 방식을 써야 할까?

### Time-based가 적합한 경우

- Hero 인트로
- 브랜드 아이덴티티 강조
- 반복 애니메이션
- "한 번만 보여주면 되는 연출"

### Scroll-based가 적합한 경우

- About/ Experience/ Works 섹션
- 사용자가 읽고 머무르는 구간
- 스크롤이 네비게이션 역할을 하는 페이지
- 사용자가 연출의 속도를 결정해야 할 때

## 6. 고급 패턴: Hybrid Animation

> 시작은 자동, 진행은 스크롤

- 섹션 진입 시 자동으로 시작되지만
- 실제 진행은 스크롤에 연동되는 방식
  최근 포트폴리오 사이트에서 가장 많이 사용되는 패턴 중 하나이다.

## 7. 한 문장으로 정리

Time-based 애니메이션은 '보여주는 연출'이고, Scroll-based 애니메이션은 '사용자에게 맡기는 연출' 이다.

## 8. 느낀 점

이 개념을 이해하고 나니
"왜 이 애니메이션이 어색한지",
"왜 여기서는 ScrollTrigger가 필요한지"를
감각이 아니라 논리로 설명할 수 있게 됐다.

애니메이션은 단순한 장식이 아니라, **UX 설계의 일부**라는 걸 다시 한 번 느꼈다.
