# SCSS 모듈 시스템: @forward 완벽 이해하기 (vs JS export)

## **1. @forward란?**

SCSS의 `@forward`는 여러 모듈(파일)을 하나로 묶어서 **외부로 다시 내보낼 때(Re-export)** 사용하는 규칙이다. 단순히 파일을 가져오는 `@use`와 달리, 모듈 간의 "통로" 역할을 수행한다.

---

## **2. JavaScript `export` vs SCSS `@forward` 결정적 차이**

### **1) "나는 못 쓰고 전달만 한다" (캡슐화)**

- **JS `export`**: 파일 내에서 변수를 정의하고 내보내면, 해당 파일 안에서도 그 변수를 자유롭게 사용할 수 있다.
- **SCSS `@forward`**: "이 기능은 내 통로를 거쳐 다른 사람에게만 가라"는 뜻이다. **`@forward`를 선언한 당사자는 정작 그 멤버(변수, 믹스인)를 직접 사용할 수 없다.**
- **Tip**: 만약 현재 파일에서도 쓰고 남에게도 전달하고 싶다면, `@forward`와 `@use`를 동시에 써야 한다.

### **2) "이름을 미리 강제한다" (Prefixing)**

- **JS `export`**: 내보낼 때 `as`를 쓰거나, 가져오는 쪽에서 이름을 결정한다.
- **SCSS `@forward`**: 통째로 **접두사(Prefix)**를 붙여서 내보낼 수 있다.

```scss
@forward "colors" as color-*;
```

- 이렇게 하면 사용하는 쪽에서 `$color-primary`처럼 쓰도록 강제할 수 있어, 여러 모듈을 합칠 때 발생하는 **이름 충돌**을 완벽하게 방지한다.

### **3) "중간 관리자" 역할 (Aggregation)**

- **JS**: `export * from './file'`을 사용하여 여러 파일을 하나로 묶는 것과 목적이 같다.
- **SCSS**: 수많은 설정 파일(색상, 폰트, 간격 등)을 하나의 `index.scss`로 묶어, 사용자에게 **"이 파일 하나만 가져다 써라"**는 식의 편리한 **단일 진입점(Entry Point)**을 제공한다.

---

## **3. 주요 기능 요약**

| **기능**        | **구문**                       | **설명**                          |
| --------------- | ------------------------------ | --------------------------------- |
| **기본 전달**   | `@forward '파일';`             | 모듈의 모든 기능을 그대로 전달    |
| **접두사 부여** | `@forward '파일' as 접두사-*;` | 이름 충돌 방지를 위해 접두사 추가 |
| **선택적 노출** | `@forward '파일' show 멤버;`   | 특정 멤버만 골라서 공개           |
| **선택적 숨김** | `@forward '파일' hide 멤버;`   | 특정 멤버만 제외하고 공개         |

---

## **4. 실전 예시: Index 패턴**

```scss
// _index.scss (중간 관리자)
@forward "colors" as color-*;
@forward "buttons" show btn-primary;

// main.scss (최종 사용자)
@use "index";

.container {
  background-color: index.$color-main; // 접두사가 붙어 안전함
  @include index.btn-primary; // 노출된 것만 사용 가능
}
```

---

## **5. 결론**

- `@use`는 기능을 **소비**하기 위해 가져오는 것.
- `@forward`는 기능을 **유통**하기 위해 전달하는 것.
- 프로젝트 규모가 커질수록 `@forward`를 통한 **네임스페이스 관리**와 **모듈화**가 필수적이다.
